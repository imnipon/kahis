Logic & Rule Summary
สรุปกฎการทำงานของ Assessment Editor
1. Rule ของ Tab
  • มี 5 tab: Tab 1-4 เป็นการประเมินแต่ละวันที่, Tab 5 เป็นตารางข้อมูลทั้งหมด
  • คลิก tab header → ซ่อน content อื่น แสดงเฉพาะ tab ที่เลือก
  • Tab ที่เลือกจะมี class active และเปลี่ยน last update by ตาม DVM ของ tab นั้น
  • Tab 1 (นิพนธ์ — คลินิกพิเศษ โรคหัวใจ นอกเวลา): editable ทั้งหมด มี badge NEW
  • Tab 2 (นิพนธ์ — อายุรกรรมทั่วไป ในเวลา): editable ทั้งหมด
  • Tab 3 (วิชัย — ภาพวินิจฉัย นอกเวลา): inactive/greyed out ทุกฟิลด์, ปุ่ม Confirm inactive
  • Tab 4 (ปนัดดา — อายุรกรรมทั่วไป ในเวลา): inactive/greyed out ทุกฟิลด์, ปุ่ม Confirm inactive
  • Tab 5 (HN 69001234 — ตารางข้อมูลทั้งหมด): แสดงตารางรวม มี filter และ search
2. Rule ของ Selected Problems/Diagnosis (Detail Grid)
  • ประเภทปัญหา (Problem Type) — เลือก: Clinical Sign (Sign), Working Diagnosis (Dx), Differential Diagnosis (DDx), Rule Out (R/O)
  • พยากรณ์โรค (Prognosis) — ไม่บังคับ: ตอบสนองการรักษาดี (Good), ต้องเฝ้าระวัง (Guarded), ความคืบหน้าไม่แน่นอน (Poor), อาการรุนแรงมาก (Grave) — ไม่แสดงเมื่อเลือก Rule Out (R/O)
  • สถานะ (Status) — เลือก: กำลังรักษา (Active), รอผลการตรวจ (Pending), หาย/สงบ (Resolved/Inactive), ต่อเนื่อง (Follow), เรื้อรัง (Chronic)
  • ความรุนแรง (Severity) — ไม่บังคับ: อาการเล็กน้อย (Mild), ปานกลาง (Moderate), รุนแรง (Severe), เสี่ยงอันตรายถึงชีวิต (Life-threatening)
  • ประเมินเวลา (Expected Duration) — ไม่บังคับ: text input
  • ระดับความมั่นใจ (Confidence) — ไม่บังคับ: มีหลักฐานสนับสนุนชัดเจน (High), มีหลักฐานบางส่วน (Medium), เป็นไปได้แต่ยังไม่ยืนยัน (Low)
  • การยืนยัน (Confirmatory Findings) — full width, ไม่บังคับ: textarea
  • เหตุผลทางคลินิก (Clinical Reasoning) — full width, ไม่บังคับ: textarea
  • หมายเหตุรายการ (Problem Note) — full width, ไม่บังคับ: textarea
  • ความสัมพันธ์ของการวินิจฉัย (Relation Diagnosis) — full width, disabled ทุก tab
3. Rule ของ Tab ที่สัมพันธ์กับ Selected Problems
  • Tab 1 (editable): ทุกฟิลด์แก้ไขได้, Problem Type ยังไม่เลือก → Prognosis disabled
  • Prognosis active เฉพาะเมื่อเลือก Problem Type = dx, ddx, หรือ Clinical Sign
  • Prognosis disabled เมื่อเลือก R/O, Resolved/Inactive, หรือ Recurrent
  • Tab 2 (editable): ทุกฟิลด์แก้ไขได้, Problem Type = dx → Prognosis active, เลือก Guarded
  • Tab 3 (readonly): ทุกฟิลด์ meta-readonly, Confirm inactive
  • Tab 4 (readonly): ทุกฟิลด์ meta-readonly, Confirm inactive
  • Tab 5 (ตาราง): รวมข้อมูลทุก tab, มี filter และ search
4. Rule ของการกดปุ่ม Close (✕)
  • ปุ่ม ✕ อยู่ที่ editor header มุมขวาบน
  • คลิก ✕ → เปิด confirm overlay เตือน "หากมีการเปลี่ยนแปลงที่ยังไม่บันทึก ข้อมูลจะหายไป"
  • ปิดโดยไม่มีการบันทึก (ปุ่มแดง) → ปิด confirm + ปิด modal ทั้งหมด
  • กลับไปแก้ไข (ปุ่มเขียว) → ปิดเฉพาะ confirm overlay
